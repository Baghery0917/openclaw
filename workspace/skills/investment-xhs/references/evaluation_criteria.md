# 五个维度评估标准

## 1. 技术维度

### 自研能力
- **评估指标**：
  - 是否有自研AI框架/算法
  - 是否在顶级会议发表论文（NeurIPS、ICML、CVPR等）
  - GitHub仓库活跃度（stars、commits、contributors）
  - 开源项目质量和技术影响力
- **数据来源**：
  - 公司官网技术博客
  - GitHub组织页面
  - 论文发布平台（arXiv、学术期刊）
  - 技术社区讨论（Reddit、知乎专栏）
- **危险信号**：
  - 完全依赖第三方API（如仅封装OpenAI API）
  - 无核心技术专利
  - 技术团队背景薄弱

### 数据壁垒
- **评估指标**：
  - 专有数据集规模和质量
  - 数据获取渠道的独特性
  - 数据标注能力和效率
  - 数据合规性（GDPR、网络安全法）
- **数据来源**：
  - 公司白皮书、技术文档
  - 行业分析报告
  - 用户协议和隐私政策

### 专利数量
- **评估指标**：
  - 发明专利数量（已授权）
  - 专利申请领域分布
  - 专利引用次数
  - 国际专利布局（PCT申请）
- **查询平台**：
  - 中国专利网（CNIPA）
  - 美国专利商标局（USPTO）
  - 欧洲专利局（EPO）
  - WIPO专利数据库

## 2. 增长维度

### ARR（年度经常性收入）
- **评估指标**：
  - 年增长率（YoY）
  - 季度环比增长率（QoQ）
  - 客户数量增长
  - 客单价变化趋势
- **数据来源**：
  - 公司财务报表（如有公开）
  - 投资者演示文稿
  - Crunchbase、PitchBook融资数据
  - 行业分析报告（IDC、Gartner）

### 用户增长
- **评估指标**：
  - 月度活跃用户（MAU）
  - 日活跃用户（DAU）
  - 用户留存率（第30日留存）
  - 用户获取成本（CAC）
- **数据来源**：
  - SimilarWeb、Alexa网站流量
  - App Store/Google Play下载数据
  - 第三方数据分析平台（Sensor Tower、App Annie）
  - 公司公开声明（如有）

### 获客成本与留存
- **计算公式**：
  - CAC = 市场营销总支出 / 新增客户数
  - LTV（客户生命周期价值）
  - LTV/CAC比值（理想值>3）
- **危险信号**：
  - CAC持续上升
  - 留存率低于40%
  - 用户流失率过高

## 3. 成本维度

### 推理成本占收入比重
- **评估指标**：
  - 计算资源消耗（GPU小时/月）
  - 云服务费用占比
  - 单位请求成本
  - 成本优化措施（模型量化、蒸馏等）
- **估算方法**：
  - 根据公开算力需求估算
  - 参考同类公司成本结构
  - 分析技术架构效率

### 算力储备
- **评估指标**：
  - 自有GPU数量
  - 云服务合同规模（AWS、Azure、GCP）
  - 算力采购长期协议
  - 能源成本控制能力
- **数据来源**：
  - 公司招聘信息（运维工程师需求）
  - 供应链合作伙伴
  - 行业媒体报道

### 毛利率
- **评估指标**：
  - 毛利率 = （收入 - 成本）/ 收入
  - 毛利率变化趋势
  - 与行业平均对比
- **行业参考**：
  - SaaS公司：70-80%
  - AI服务公司：50-70%
  - 硬件公司：30-50%

## 4. 竞争维度

### 差异化优势
- **评估方面**：
  - 技术独特性
  - 商业模式创新
  - 市场定位精准度
  - 品牌影响力
- **分析方法**：
  - SWOT分析（优势、劣势、机会、威胁）
  - 波特五力模型
  - 竞品功能对比矩阵

### 客户切换成本
- **评估指标**：
  - 数据迁移难度
  - API兼容性
  - 集成工作量
  - 合同约束条款
- **危险信号**：
  - 产品同质化严重
  - 无长期合同约束
  - 标准化接口易替换

### 巨头覆盖风险
- **风险因素**：
  - 大公司类似功能发布时间
  - 资源碾压可能性
  - 生态锁定效应
  - 价格战承受能力
- **防御能力**：
  - 专利保护强度
  - 客户关系深度
  - 垂直领域专业化

## 5. 团队维度

### 核心人员背景
- **评估方面**：
  - 创始人行业经验
  - CTO技术背景（论文、开源贡献）
  - 核心团队完整性（技术、产品、销售）
  - 顾问委员会影响力
- **数据来源**：
  - LinkedIn个人资料
  - 学术论文作者身份
  - 行业会议演讲记录
  - 媒体报道专访

### 工程落地经验
- **评估指标**：
  - 过往创业/项目成功率
  - 产品化能力证明
  - 大规模系统架构经验
  - 客户交付记录
- **危险信号**：
  - 纯学术背景无产品经验
  - 团队缺乏工程人才
  - 过往失败记录过多

### 人才结构
- **理想比例**：
  - 技术研发：40-60%
  - 产品设计：10-20%
  - 市场营销：15-25%
  - 运营支持：10-20%
- **评估要点**：
  - 关键岗位是否有空缺
  - 员工流失率
  - 招聘难度和速度
